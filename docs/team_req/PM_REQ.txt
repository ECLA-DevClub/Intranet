Регламент кибербезопасности для Project Manager
1. Ответственность

PM отвечает за:

внедрение security-практик

контроль выполнения регламентов

управление рисками

2. Управление рисками

Вести Risk Register

Классифицировать:

High / Medium / Low

Примеры:

утечка данных

компрометация SSO

XSS/SQLi

3. Планирование безопасности

В каждый спринт включать:

security tasks

bug fixing

обновления зависимостей

4. Контроль доступа

Назначение ролей:

Dev / QA / Admin

Контроль:

кто имеет доступ к продакшену

Регулярный аудит доступов

5. Управление инцидентами

PM обязан:

определить Incident Response Plan

обеспечить:

время реакции (SLA)

коммуникацию

При инциденте:

Изоляция

Анализ

Устранение

Отчёт

6. Контроль требований безопасности

Из документа:

TLS обязателен

защита от XSS/CSRF/SQLi

uptime 99.9%

PM контролирует:

выполнение этих требований

7. Работа с подрядчиками

Проверка:

security compliance

Подписание:

NDA

security agreements

8. Обучение команды

Регулярно:

security training

Минимум:

OWASP Top 10

phishing awareness

9. Управление релизами

Запрещено:

выкатывать без security проверки

Обязательно:

staging тестирование

10. Документация

PM должен поддерживать:

Security Policy

Incident Logs

Access Logs
