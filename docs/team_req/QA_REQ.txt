Регламент кибербезопасности для тестировщиков 
1. Общие обязанности

Проверка системы на уязвимости

Валидация требований безопасности

2. Тестирование уязвимостей
Обязательно проверять:

XSS

CSRF

SQL Injection

IDOR (Insecure Direct Object Reference)

Broken Auth

3. Тестирование аутентификации

Проверить:

SSO корректность

logout

session expiration

Попытки:

brute force

bypass auth

4. Тестирование RBAC

Проверить:

доступы пользователей

доступ админа

Нельзя:

получать доступ к чужим данным

5. API тестирование

Проверять:

авторизацию

rate limiting

доступ к endpoints

Инструменты:

Postman

Burp Suite

6. Тестирование загрузки файлов

Проверить:

загрузку вредоносных файлов

обход ограничений

Попытки:

загрузить .php/.js

7. Тестирование фронтенда

Проверка:

XSS через формы

утечка токенов

Проверить:

CSP

8. Логирование и аудит

Проверить:

логируются ли события

нет ли утечек

9. Автоматизация

Использовать:

OWASP ZAP

Burp Scanner

Интеграция в CI/CD

10. Отчётность

Каждая уязвимость:

описание

шаги воспроизведения

уровень риска

рекомендации
